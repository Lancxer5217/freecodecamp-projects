# freecodecamp-projects

A living repository for my journey through the FreeCodeCamp curriculum.

## Purpose

This repository serves two interconnected goals:

1. **Learn Git & GitHub**  
   This is my dedicated workspace for mastering version control. Every project here is an opportunity to practice the core Git workflow: `add`, `commit`, `push`, and writing conventional commit messages. The goal is to build an intuitive understanding of how Git tracks code changes over time.

2. **Learn via FreeCodeCamp courses and track my own progress**  
   This repo acts as a central portfolio and progress tracker for all my certification work. It allows me to visually see how far I've come as I move through the curriculum and serves as a record of my growth as a developer.

## Git Learning Goals

The following practices are being actively applied in this repository:

- **Atomic commits** – Each commit represents one logical change (e.g., passing a related group of user stories).
- **Conventional Commits** – Using standard prefixes (`feat`, `fix`, `style`, `chore`, `docs`) to keep the history readable and automated.
- **Remote collaboration** – Pushing commits to GitHub regularly to simulate a real-world remote workflow.
- **Clean history** – Using `git status` before commits and `git log --oneline` to review the timeline.
